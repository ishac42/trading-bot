# Trading Bot - UI/UX Design Document

## Design Philosophy
- **Clarity First**: Trading data must be clear and unambiguous
- **Real-time Updates**: Live data is critical for trading decisions
- **Action-Oriented**: Quick access to bot controls (start/stop/pause)
- **Data-Rich**: Show all relevant information without clutter
- **Professional**: Financial-grade UI that inspires confidence

### Components
1. **Summary Cards** (Top Row)
   - Total P&L (with percentage change)
   - Active Bots count
   - Open Positions count and value

2. **Active Bots Section**
   - Card for each bot showing:
     - Bot name
     - Status badge (Running/Paused/Stopped)
     - Action buttons (Pause/Resume/Stop)
     - Symbols being traded
     - Capital allocated
     - Current P&L
     - Indicators used
     - Trades today count
     - Win rate
   - "Create New Bot" button

3. **Recent Trades Table**
   - Last 5-10 trades
   - Columns: Time, Symbol, Type (Buy/Sell), Quantity, Price, Bot, P&L
   - Link to full trade history

### Interactions
- Click bot card â†’ Navigate to bot details
- Click trade row â†’ Show trade details modal
- Click "Create New Bot" â†’ Navigate to bot creation
- All data updates via WebSocket

### Components
1. **Header**
   - Page title "Bots"
   - "Create New Bot" button (primary action)

2. **Filters & Search**
   - Status filter tabs (All, Running, Paused, Stopped)
   - Search bar for bot names

3. **Bot Cards**
   - Each bot in a card showing:
     - Bot name
     - Status badge with color coding
     - Action buttons (context-aware: Edit, Start/Stop/Pause/Resume, Delete)
     - Symbols being traded
     - Capital allocation
     - Indicators configuration
     - Trading window
     - Performance metrics (P&L, trades count, win rate)
     - Last trade timestamp

## Component Library

### Reusable Components

1. **StatusBadge**
   - Running (green)
   - Paused (yellow)
   - Stopped (gray)
   - Error (red)

2. **P&LDisplay**
   - Shows amount and percentage
   - Color: Green (positive), Red (negative)
   - Format: +$1,234.56 (+2.5%)

3. **BotCard**
   - Reusable card for bot display
   - Shows all bot info
   - Action buttons

4. **TradeTable**
   - Sortable, filterable table
   - Real-time updates
   - Pagination

5. **PriceChart**
   - TradingView Lightweight Charts wrapper
   - Buy/sell markers
   - Indicator overlays

6. **IndicatorConfig**
   - Reusable form component for indicator setup
   - Shows/hides parameters based on selection

7. **RiskInput**
   - Input with label, tooltip, and validation
   - Percentage inputs with visual feedback

## User Flows

### Flow 1: Create and Start a Bot
1. User clicks "Create New Bot" (Dashboard or Bots page)
2. Fill out bot creation form
3. Click "Save Bot"
4. Bot appears in list with "Stopped" status
5. User clicks "Start" button
6. Bot status changes to "Running"
7. Bot begins trading (trades appear in real-time)

### Flow 2: Monitor Trading Activity
1. User on Dashboard
2. Sees active bots with live P&L
3. Recent trades table updates in real-time
4. Price chart shows latest prices
5. User clicks on a trade â†’ See trade details
6. User clicks on a bot â†’ See bot details/statistics

### Flow 3: Manage Positions
1. User navigates to Positions page
2. Sees all open positions with live prices
3. Position P&L updates in real-time
4. User clicks position â†’ See detail panel
5. User can manually close position if needed

### Flow 4: Analyze Performance
1. User navigates to Trades page
2. Applies filters (date range, bot, symbol)
3. Reviews trade history
4. Views analysis charts
5. Exports data to CSV

## Design System

### Colors
- **Primary**: Blue (#1976d2) - Actions, links
- **Success**: Green (#4caf50) - Profits, positive values
- **Warning**: Orange (#ff9800) - Paused, warnings
- **Error**: Red (#f44336) - Losses, errors, stop actions
- **Neutral**: Gray (#757575) - Stopped, inactive
- **Background**: Light gray (#f5f5f5)
- **Surface**: White (#ffffff)

### Typography
- **Headings**: Bold, 16-24px
- **Body**: Regular, 14px
- **Small**: 12px for labels, timestamps
- **Monospace**: For numbers, prices, codes

### Spacing
- **Card padding**: 16px
- **Section spacing**: 24px
- **Component spacing**: 8px, 16px, 24px

### Icons
- Use Material Icons or similar
- Status indicators: Circle icons with colors
- Actions: Standard icons (play, pause, stop, edit, delete)

## Real-time Updates Strategy

### WebSocket Events
1. **Trade Executed**: Update trades table, bot P&L, dashboard summary
2. **Position Updated**: Update position prices, P&L
3. **Bot Status Changed**: Update status badges, action buttons
4. **Price Update**: Update charts, position prices
5. **Market Status**: Update market status indicator

### Update Frequency
- Prices: Every second during market hours
- Positions: On every price update
- Trades: Immediately when executed
- Bot status: On state changes

### Visual Feedback
- Highlight new trades (flash animation)
- Update numbers with smooth transitions
- Show connection status indicator
- Auto-reconnect on disconnect
